# 灵创健康网站 Roadmap

这份路线图用于描述当前项目已经完成的能力、下一阶段建议优先推进的内容，以及面向生产环境仍需补足的部分。

## 当前已完成

### 站点与页面

- 官网首页已完成
- 新闻列表与新闻详情页已完成
- 产品详情页已完成
- 招聘页已完成
- 管理后台页面已完成

### 前端体验

- 首页 banner 切换效果已完成
- 多页面前端构建已完成
- GitHub Pages 静态发布配置已完成
- 本地构建与 Pages 构建路径已分离

### 后端与数据

- Express 后端已接入
- SQLite 数据库已接入
- 产品、新闻、留言相关 API 已接入
- 基础后台登录与内容管理接口已接入
- 留言频率限制已接入

## 下一阶段建议优先做

### 1. 后台能力补全

- 增加首页配置管理，让轮播图、指标、解决方案等内容可在后台直接编辑
- 为招聘页增加岗位管理，而不是只展示静态内容
- 增加图片上传能力，避免后台只能手填图片 URL

### 2. 发布链路补全

- 为前端静态站点之外的后端服务补一套真实部署方案
- 明确线上 API 地址和环境变量策略
- 区分开发、测试、生产三套配置

### 3. 安全与账号体系

- 移除硬编码管理员密码
- 改为基于数据库的管理员账号体系
- 使用正式 token / session 方案替代演示鉴权
- 为关键写操作补权限校验和错误处理

## 中期演进方向

### 内容与运营

- SEO / TDK 优化
- 新闻分类、标签、分页与检索
- 产品内容的结构化管理
- 留言通知能力，例如邮件或企业微信通知

### 体验与视觉

- 深色模式进一步完善
- 滚动入场动画与更完整的动效系统
- 移动端导航与交互继续打磨
- 骨架屏、加载态和空状态统一

## 面向生产仍需补足

- GitHub Pages 只能部署前端静态资源，当前后端还未形成正式线上部署
- `server/database.sqlite` 仍是本地文件型数据库，不适合更高并发场景
- 后台鉴权仍是演示实现，不适合真实生产环境
- 还没有自动化测试体系
- 还没有完整的 CI/CD 校验链路

## 建议的近期执行顺序

1. 后台首页配置管理
2. 后端正式部署方案
3. 管理员鉴权重构
4. 新闻 / 招聘 / 产品内容管理继续细化
5. SEO 与生产级监控补齐

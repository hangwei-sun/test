# 灵创健康 (Lingchuang Health) 网站迭代演进路线图

为了将当前的初创版本打造成一个**企业级、高可用、体验极致**的现代化官方网站，我为您规划了以下四个阶段的渐进式迭代计划。每个阶段都聚焦于解决特定的业务或技术痛点。

---

## 阶段一：内容全面管理与 SEO 优化 (基础夯实)

目前网站的“产品”和“留言”已经可以动态管理，但主页的其他文案（如轮播图、解决方案）在后台还需要界面支持。同时，企业官网对搜索引擎的收录（SEO）要求极高。

- [ ] **全站内容管理**：在管理后台 (`/admin.html`) 增加“首页配置”模块，允许管理员无代码修改首页的轮播图、核心数据指标、解决方案等所有板块的文字与图片。
- [ ] **图床集成 (OSS)**：在后台编辑器中增加图片上传功能，将图片自动传至阿里云 OSS 或腾讯云 COS，代替手动输入图片 URL。
- [ ] **SEO 与 TDK 优化**：为每个产品详情页动态生成 `<title>`, `<meta name="description">` 和 `<meta name="keywords">`。
- [ ] **SSG / 预渲染引入**：针对当前纯客户端渲染（CSR）带来的 SEO 弱势，引入 Vite-SSG 或将其轻量化迁移至 Next.js / Nuxt，确保百度和 Google 等爬虫能完美抓取页面内容。

## 阶段二：视觉体验与交互升级 (品牌质感)

在已有美观的基础上，进一步提升“数字医疗生态”的科技感和品牌信任度。

- [ ] **滚动视差与入场动效**：引入 `IntersectionObserver` 或 `GSAP`，当用户向下滚动网页时，模块内容如丝般顺滑地“浮现”或“错位滑动”（Scroll Reveal），大幅提升高级感。
- [ ] **深色模式 (Dark Mode)**：开发全站的深浅色主题切换能力，满足现代操作系统规范，并自动适应用户的系统偏好。
- [ ] **移动端体验重构**：将当前的顶部导航栏在手机端重构为优雅的侧滑抽屉菜单 (Drawer)，优化触屏下的轮播图滑动体验。
- [ ] **骨架屏与加载状态**：在请求后端数据时，使用闪烁的骨架屏（Skeleton Screen）替代生硬的“数据加载中”文字。

## 阶段三：业务模块与场景扩展 (功能丰满)

随着企业发展，官网需要承担更多业务触达和营销转化的责任。

- [x] **新闻与洞察专栏**：开发“动态/博客”系统，支持文章分类、分页检索、标签管理。让贵公司能持续输出行业洞察，进行内容营销。
- [x] **招聘与企业文化**：新增 `/careers` 页面，展示团队风采并对接开放职位，后台可直接管理招聘岗位与收取简历。
- [x] **留言板反垃圾与通知**：
  - 为留言板接入极验或验证码，防止机器人恶意刷接口。
  - 留言提交后，自动触发邮件通知或企业微信/钉钉机器人通知管理员。
- [x] **地图接入**：在页脚或联系我们板块，嵌入高德/百度地图 SDK，直观展示公司位置。

## 阶段四：架构重构与安全加固 (面向生产)

当网站流量上升，我们需要保证系统的绝对稳定与安全。

- [ ] **权限与安全隔离**：
  - 废弃硬编码密码，重构为基于数据库的管理员账号体系。
  - 采用标准的 JWT (JSON Web Token) 进行鉴权，设置 Token 过期时间与刷新机制。
- [ ] **数据库升级**：将 SQLite 结构无缝迁移至 MySQL 或 PostgreSQL，并引入 Redis 作为热点数据的缓存层（如首页配置数据），极大地提高并发访问能力。
- [ ] **前端工程化规范**：如果业务继续膨胀，可将原生的 Vanilla JS 逐步重构为 React 或 Vue3 组件化架构，引入 TypeScript 规范数据接口定义，避免运行时报错。
- [ ] **CI/CD 自动化部署**：完善 `.github/workflows`，实现代码 Push 到 `main` 分支后，自动进行构建、测试并部署到云服务器（如 Docker 化部署）。
